importEpub(epubPath, metadataopt, importOptionsopt, callbackopt) → {IntellogoResponse.<ImportAPI~EpubImportResult>}

Import a book in an .epub format.
The book is automatically split in parts and chapters per the epub's ToC. Intellogo will create an entity for each chapter of the book. Depending on the TOC, if the book was split e.g. in parts that in turn contain chapters, there will also be separate entities for each part (containing text from their assigned chapters). The process is recursive, so the result will consist of a hierarchy with the same depth as the book's ToC. In all cases, there will be a content representing the whole book.
Metadata for the imported content is parsed from the epub's metadata. If additional metadata is specified, the epub extracted one is overwritten.
If chunking is enabled, the ToC structure will be ignored and the whole text of the book will be split into equally-sized chunks, as in ImportAPI#importArticle. This will result in a one-level deep hierarchy for the book.
If the import is successful, the ID of the content for the whole book can be found by looking at the last element of the importSummary array in the result.

importSubtitles(subtitlesPath, metadata, importOptions, callbackopt) → {IntellogoResponse.<ImportAPI~ArticleImportResult>}

Import an .srt file containing timestamped subtitles. Metadata must be supplied, there is no automated metadata extraction for .srt files.
Intellogo supports two modes of chunking for .srt files:

  • length-based chunking: This is the standard chunking, like the one for articles. The text content of the subtitles is split into equally-sized chunks.
  • timestamp-based chunking: If a timestamp separator file is provided, each chunk will contain the captions with timestamp between consecutive lines of the timestamp separator file. This allows splitting the subtitles in a more semantic way - e.g. for a documentary's subtitles the user can manually specify the timestamps at which a new topic begins.

SubtitlesImportOptions

Type:
  • Object
Properties:
Name Type Attributes Description
chunkingMode String <optional>

What chunking to apply to the imported subtitles. Allowed values: length-based or timestamp-based
chunkSize Number <optional>

Length (in characters) of each chunk.
Required if chunkingMode is length-based.
timestampsFile String <optional>

The path to a file specifying timestamps on which to split the subtitles. This must be a text file. Every line specifies a splitting point in the captions in the format hh:mm:ss,ms <ENTRY NAME>. Every splitting point will create a new content chunk containing all captions between the previous line and this line's timestamp. ENTRY NAME will be used as title in the chunk's metadata.
For example, a timestamp file with two lines

00:15:00,000 First Half
00:30:00,000 Second Half

will create two chunks from the subtitles file - one with all captions from the first 15 minutes and one with all captions from 00:15 until 00:30 minutes into the subtitles. If the subtitles file contains captions after the 30th minute, they will not be included in any chunk (although they will be used when importing the full content).
Required if chunkingMode is timestamp-based.
